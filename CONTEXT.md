# CONTEXT

本仓库的领域术语表(glossary)。只定义"词的意思",不放实现细节。

## 经验沉淀家族(Sediment family)

一套"把会话里摸索出的 know-how 固化下来"的机制,受地质隐喻统领:经验像沉积物,层层堆积、压实,达到时间+压力阈值才"成岩"、进入永久记忆。

### Sediment(沉积 / 复盘顾问)

已存在的 skill。**手动**调用的复盘顾问:回顾**当前会话**,扫描四类信号(重复指令 / 解决的难题 / 反复纠正 / 明确偏好),对每个候选判断该沉淀到哪(skill / CLAUDE.md / rule / 不固化)。

它是**无状态**的:只看活着的这次会话,**不写任何文件**,只给建议。这是它的洁癖,不改。

### Sediment-Basin(沉积盆地 / 累积器)

新增 skill。承接 sediment 家族里的"脏活":把候选经验**持久堆积**进一个跨会话的**盆地**,做去重、计数,达阈值才晋升。与 sediment 的关系同构于参考里 self-improvement / self-healing 的分工——顾问只诊断,盆地才落库、改文件。

### 盆地(Basin / 持久池)

Sediment-Basin 的存储层。**每个项目各一个**,存放该项目积累的沉积物条目;晋升时写进**该项目自己的** CLAUDE.md。跨项目的通用偏好不在此层的职责内。

### 沉积物条目(Sediment entry)

盆地里的一条记录。一条候选经验带上元数据(Pattern-Key、复发次数、首见/末见时间等),供去重与计数。

### Pattern-Key(模式键)

沉积物条目的**稳定去重键**。判断"这次的经验是不是上次那条的复发",靠比对 Pattern-Key 而非字面文本。相同 Pattern-Key 归为同一堆,累加复发次数。

命名走 `域.slug`(如 `voice.no-hype`、`ppt.center-lines`)。纪律是**先搜后铸**:铸新 key 前先在盆地里搜现有 key,语义命中就复用+复发+1,没命中才新建——这道纪律才是去重真正生效的地方,否则同一坑被打成两个 key,复发永远攒不到阈值。

### 复发(Recurrence)

同一 Pattern-Key 的经验再次出现。每复发一次,该条目的复发计数 +1,并刷新末见时间。

### 成岩阈值(Lithification threshold)

一条沉积物"够格晋升"的量化门槛。三条**同时**满足:复发计数 ≥ 3、跨 ≥ 2 个不同任务、且都落在 30 天窗口内。取自参考 skill 的单一晋升规则。

### 晋升(Promotion)

把够格的沉积物,蒸馏成一条**简短的"预防规则"**(写"以后编码前/中该怎么做",不搬事故长文),写进目标项目的 CLAUDE.md。够格有两条路:①**自动够格**——达到成岩阈值后冒泡、经人点头;②**手动放行(Manual override)**——人一眼看中某条,不必等凑满阈值,直接拍板转正。晋升后条目状态标记为已晋升。

晋升去向有两个:**CLAUDE.md**(一句预防规则),或当经验反复且通用时,**抽成一个可复用 skill**(对应老 sediment 的"做成 skill"分支)。

### 成岩阈值可调(Tunable threshold)

复发≥3、跨2任务、30天窗口这三个数字是**默认值、可改**。单人/单项目场景嫌太严,可自行调松。

### 条目状态(Entry status)

每条沉积物带一个状态:`待处理` → `已晋升` / `已丢弃` / `已抽成skill`。用来查"还有哪些没处理",避免重复冒泡已处理过的条目。

### 账本视图(Ledger view)

盆地对人可读的一面:一份**方便翻看的文档**,列出当前所有沉积物条目及其复发次数、状态,让人一眼看清攒了些什么、哪些快成岩了。

### 月度复盘提示(Monthly review nudge)

每隔约 30 天,提醒人来翻一次账本视图、处理待处理条目、手动放行值得的经验。靠比对"上次复盘日期"实现,不需要额外定时基础设施。

### 任务(Task)

"跨 ≥ 2 个任务"里的计数单位,取 **commit**:一个 commit SHA = 一个任务。同一 Pattern-Key 在 ≥2 个不同 commit 上出现,才算"跨任务"。hook 在 commit 后触发,天然知道当前 SHA,无需额外机制。
