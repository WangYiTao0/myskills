# Milwaukee Tool 简报美化规范

适配自 willem4130/elite-powerpoint-designer（2.4K 装机）+ Tufte 数据可视化原则 +
本品牌模板的几何约束。所有数字都是**针对 Milwaukee 16:9 模板**调过的，不是
照搬 keynote 风格。

## 几何约束（先记住这个）

模板：33.87 × 19.05 cm，16:9。

```
0───────────────────────────── 33.87 cm ───────────────────────────────
│  [Milwaukee logo]   ████ RED BANNER ████   [Title 24pt | Subtitle 15pt]
│                                                                          ← 0–1.0 cm  锁定
├──────────────────────────────────────────────────────────────────────
│
│             可用内容区  31.87 × 16.0 cm
│             left=1.0  top=1.5  right=32.87  bottom=17.5
│
├──────────────────────────────────────────────────────────────────────
│  Confidential Document Property of MILWAUKEE TOOL ...                    ← 18.0–19.05 cm 锁定
└──────────────────────────────────────────────────────────────────────
```

**只能动中间这块。** 红色横幅和页脚都在 slideMaster，碰了就破坏品牌一致性。

## 字号阶梯

| 用途 | 字号 (pt) | 字重 | 行高 |
|------|----------|------|------|
| 主标题（红色横幅，模板锁定） | 24 | Bold | — |
| 副标题（红色横幅，模板锁定） | 15 | Regular | — |
| 内容区一级标题 | 28-32 | Bold | 1.2 |
| 内容区二级标题 | 20-24 | Semibold | 1.3 |
| 正文（默认） | **16** | Regular | 1.4 |
| 正文（密集场景） | 14 | Regular | 1.4 |
| 表格表头 | 12 | Bold | 1.2 |
| 表格正文 | 11 | Regular | 1.3 |
| 标注/来源 | 10-11 | Regular Italic | 1.3 |

中文（zh-TW）在同 pt 下视觉偏小，比英文阶梯下调 1–2pt 也可以接受；但**一篇 deck
内不要混阶梯**。

**最多 2 个字体族**：Latin 用 Calibri，CJK 用 Microsoft JhengHei（Mac 回退
PingFang TC / Heiti TC）。不要 Arial、Times、Comic Sans 之类乱入。

## 配色系统

```
Milwaukee 红     #DB021D    强调色，每页面积 ≤ 10%
深字色          #333333    正文
中灰            #666666    标注、说明、来源
浅灰            #F5F5F5    斑马行、卡片底
白              #FFFFFF    背景
```

状态色（仅用于 KPI / 图表，不用于排版装饰）：

```
成功 #2E7D32   警示 #ED6C02   危险 #C62828
```

**禁用：**渐变、3D 效果、文字阴影（标题阴影是模板自带的，别再加）、装饰边框、
彩虹色板。

## 间距网格

```
边距             1.0 cm  （≥ 这个，否则视觉太挤）
段落之间         0.3 cm
小节之间         0.6 cm
项目符号缩进     0.95 cm  （= 342900 EMU，模板内置）
列间距（多列）   0.5 cm
卡片内 padding   0.4 cm
```

## 内容密度（6×6 规则改良版）

- 一页主题 1 个，不要"杂烩页"
- 项目符号：**最多 6 条 / 页，每条 ≤ 12 字（中文）/ ≤ 6 词（英文）**
- 正文段落：**最多 6 行**
- 表格：**最多 7 行**（含表头），列数 ≤ 6
- 演讲节奏：约 **1 页 / 分钟**

超量时拆页或归并到附录。

## 对比度 & 可读性

- 正文 vs 背景对比度 ≥ **4.5 : 1**（WCAG AA）
- 红底白字（标题区）已通过；不要在红底上放小字
- 标题不要超 40 字符（横幅会截）；副标题不要超 60 字符

## 图片规范

- 最低 1280×720，**首选 1920×1080+**
- 产品图：白底或透明背景，Milwaukee 官方拍摄优先，不要裁切到边缘外
- 截图：PNG，UI 元素清晰；视频帧不要直接用
- 不用 clipart、不用 emoji 当装饰

## 图表（Tufte 化的 data-ink 原则）

- 优先 2D **柱状图 / 折线图**；饼图最多 4 片，超过改柱状
- **直接在数据上打标**，少用图例
- 网格线浅灰即可，不抢戏
- 一个图一个故事，不要塞两个 Y 轴
- 数据来源标在右下角 10pt 灰字

## 版式套路（适配模板）

| 用途 | 推荐布局 |
|------|----------|
| 章节封面 | 内容区只放一个超大居中红字小节名（48-60pt）+ 一行副标 |
| 重点结论 | 单大字 KPI（72pt 红）+ 下方一行说明 |
| 三列对比 | 三等分列，每列顶图 + 顶标题 + 短描述 |
| 时间线 | 横向 5 节点，节点用红色实心圆 |
| 引述 | 居中大字斜体 + 来源右对齐小字 |
| Q&A 收尾 | "QUESTIONS?" 大字 + 联系方式 |

## 反模式（直接砍）

- ❌ 在内容区再加红色块（破坏横幅的视觉锚点）
- ❌ 主标题用句子（应该 1-3 词大写英文）
- ❌ 同一页放 logo（已在 master 里）
- ❌ 多种项目符号样式混用
- ❌ 装饰性图标 / clipart
- ❌ 全大写正文（仅标题用）
- ❌ 中英文之间不留空格（"Milwaukee红"应是"Milwaukee 红"）
