# 版式目录（实战沉淀的案例参考）

## 使用定位

本目录是**已验证的案例参考，不是版式白名单**：

- 内容与某个案例匹配 → 直接复用，省去现场设计
- 内容不匹配 → **自行设计版式**，不要硬套案例。自设版式只需守住三件事：
  1. SKILL.md §6：品牌锚点 + 色板 + 红色纪律
  2. SKILL.md §7：内容硬约束（一节一信息、禁用元素）
  3. ppt-design-principles skill：通用设计原则（对齐 / 对比 / 重复 / 亲密性）
- 部分案例标注了**适用前提** —— 前提不成立就自行设计，不要为了套版式硬凑内容

来源：A 系列来自一份 15 页项目复盘汇报（Agenda + PDCA 完整结构）；B 系列来自一份 17 页月度工作总结。
所有描述已脱敏，不含真实数据。红色用法均受 SKILL.md §6.3 红色纪律约束。

## A. 项目汇报版式

### A1 · 全红封面 Cover

- **用于**：Page 1
- **结构**：模板自带全红底 + 白色 logo + tagline；项目主标题 + 副标题（项目阶段，如 Post-Launch Review）+ 汇报人/日期一行
- **红色**：模板自带，不再叠加任何元素

### A2 · Agenda 双组八宫格

- **用于**：目录页
- **结构**：8 节按"项目定义（01–04）/ 执行与沉淀（05–08）"分上下两组；每节一张小卡片 = 线性 icon + 两位编号 + 中文标题 + 英文标题
- **红色**：编号或 icon 单色红，卡片底白/浅灰

### A3 · 战略锚定 + 痛点页（BACKGROUND）

- **适用前提**：项目能明确承接公司/部门级战略。无战略锚定的项目不要硬凑——背景页直接用痛点 icon row 全页展开，或自行设计
- **用于**：§01 项目背景
- **结构**：上半 = 公司战略支柱横排卡片，**本项目承接的那一柱红框标出**；下半 = 痛点 icon row（01/02/03 + 大数字佐证）
- **红色**：战略支柱只红一个；痛点只给风险最高的一个点红（§6.3 问题页判据）

### A4 · 三阶段递进页（TARGET）

- **适用前提**：目标有"现状 → 本阶段 → 长期"的阶段演进。单一目标的项目用大数字 / 目标卡片直接陈述即可
- **用于**：§02 项目目标
- **结构**：横向三栏 = 现状 → 本阶段（中央强调卡）→ 长期；每栏标题 + 3 条说明
- **红色**：仅中央"本阶段"卡片用红色强调（顶条或标题）

### A5 · 范围三卡页（SCOPE）

- **用于**：§03 项目范围
- **结构**：01/02/03 三张数字卡片并排，每卡 = 编号 + 标题 + 包含项清单；可加"不包含"边界说明行
- **红色**：编号圆点首个用红或全灰，≤1 处

### A6 · 大数字 + 列表页（USE CASES）

- **用于**：§04 应用场景（或任何"一个总量 + 多个分项"内容）
- **结构**：左侧超大红数字（60–80pt，总量），右侧 01/02/03 分项列表
- **红色**：只有左侧大数字

### A7 · 时间线页（MILESTONE）

- **用于**：§05 项目节点
- **结构**：横向主线（`#E5E5E5`）+ 节点圆点 + 节点上下交替放标签；底部可加 1–2 个 stat callout
- **红色**：关键节点（kick-off / 上线）红点，其余灰点

### A8 · Stat 墙成果总览（OUTCOMES 总览）

- **用于**：§06 首页（成果总览）
- **结构**：上半 = 3–4 个大数字 stat callout 并排；下半 = 分轨道明细（双轨如软件/硬件；单轨交付就单列，不硬拆）
- **红色**：**只染 1 个最该被记住的数字**，其余深灰，大小造对比

### A9 · 实景 Before / After 页

- **适用前提**：存在可拍摄的物理变化且有改造前的留底照片。纯软件/流程类成果改用 A10 或 B3
- **用于**：§06 物理改造类成果（场地、设备）
- **结构**：左 BEFORE 实拍 / 右 AFTER 实拍 + 中间红箭头 + 底部一行结论
- **红色**：箭头 + AFTER 侧标注；BEFORE 永远灰
- **配图**：实拍照片必须用户提供

### A10 · 系统截图页

- **用于**：§06 软件/系统类成果
- **结构**：大截图占主体（白底细灰边框）+ 侧栏 feature 列表（icon row）+ 入口说明
- **红色**：侧栏首项红圆编号，截图内不加红框（除非标注关键区域，≤1 处）
- **配图**：真实系统截图必须用户提供

### A11 · 极简流程页

- **用于**：§06 数据流/架构说明
- **结构**：横向节点流程 = 圆形/方形节点 + 细线连接 + 节点下注释；禁 SmartArt
- **红色**：仅关键节点（如核心平台）1 处

### A12 · Org Chart 团队页（TEAM）

- **适用前提**：团队有明确的 owner + 层级结构。扁平协作或跨部门虚拟团队改用分工卡片（成员 + 职责并列），不画层级树
- **用于**：§07 项目团队
- **结构**：顶层 owner 节点 + 下层成员节点 + 底部 01/02/03 分工说明
- **红色**：仅 owner 节点
- **注意**：真名仅限内部版本，对外场景用角色名

### A13 · 新增能力三卡页（IMPROVEMENT）

- **用于**：§08 上线后已落地的迭代（`IMPROVEMENT` 只用作 08.x 子节点英文标题——§1 允许子节点英文自由；第 08 节父标题仍固定为 `LESSONS`）
- **结构**：三张截图卡片并排，每卡 = 编号 + 截图 + 标题 + 一句说明
- **红色**：编号首个红，其余灰

### A14 · Lessons 编号表格页

- **用于**：§08 经验沉淀
- **结构**：编号列表 5 行以内，每行 = 编号 + 经验标题 + 一句展开 + 可迁移项目标注
- **红色**：**最重要的一行浅红底 `#FCE6E8`**，其余白底

### A15 · 实景收尾页（Closing）

- **用于**：最后一页
- **结构**：现场实拍大图为底 + 角落大字结论（核心 stat 一句，如稳定运行时长 + 服务人数）+ 署名
- **红色**：照片自带环境红即可，不叠红色块
- **配图**：实拍照片必须用户提供

## B. 月报版式

月报与项目汇报的根本差异：**月报按"工作项"组织，每页一项，证据优先**；不走 Agenda 八节。

### B1 · 月报封面

- **结构**：模板全红 Cover + `[Month]. [Year] Working Summary` + 日期 / 姓名 / Update

### B2 · 工作证据页（月报主力版式，约 80% 页面）

- **结构**：
  - banner 主标题 = 本月主题线（一条主题线可连续多页），副标题 = 本页具体子项
  - 正文区 = 截图 / 文档 / 表格等**实物证据**为主体，少量标注文字
  - **底部红条**（每页必有）= `✔ 已完成结论一句` + `Next: 已确定的下一步`
- **红色**：底部红条为版式自带（§6.3 配额豁免）；正文区内仍按 ≤5% 执行
- **要点**：一页一个工作项；证据 > 文字；底部结论是这页唯一必读句

### B3 · Before → After 转换页

- **用于**：月内流程/方法改进
- **结构**：左侧旧状态块（深灰/黑底）→ 红箭头 → 右侧新流程节点横排（红底白字小块）
- **红色**：新流程侧；旧状态永远深灰

### B4 · Key Results 面板页

- **用于**：月内某主题的成果小结
- **结构**：左列 = 红框 Key Results 列表（标题 + 一句说明，逐条），右侧 = 对应证据截图 2–3 张
- **红色**：左列红框描边 + 列表标题，截图区不加红

### B5 · Thank You 收尾

- **结构**：模板自带红色 Closing 页，不加内容
