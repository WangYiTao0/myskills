# 分享课件文案纪律

为培训课件、内部分享、讲义类文本把关。目标:**教方法不秀成果,尊重听众,每个论点都有例子垫着。**
来源:2026-06《数据思维与呈现》课程逐页打磨中沉淀的六条铁律。

## 一、不吹自己

- 课程以**通用方法论为骨架**;自己的项目至多作少量配图佐证,不做"案例全解剖"
- 禁炫耀性统计当卖点("我这张报表 XX 页 / XX 个度量"式数字名片)
- 讲概念时不反复回指"我的报表如何如何";配图不报户口、不讲幕后工程

## 二、不贬听众

- 不写**假设听众出错**的句子:✗"为什么你的表 BI 用不了" ✗"跳过这一步,KPI 必吵架" ✗"使用者悄悄修数"
- 批评只指向**匿名演示样例**(一张虚构的坏表),永不指向听众或同事
- 冲突主体写成"数字 / 机制",不写成"人":✗"两个部门吵" → ✓"两个数字对不上,差的是口径"
- 不用"只提这三个要求"式的居高临下句;规则写给"表",不写给"人"

## 三、正文不放舞台调度语

- 幻灯片正文只放内容主张;调度话术留给讲者口播
- 禁:✗"诊断先挂起,回来收它" ✗"接下来 N 分钟讲 X" ✗"今天只讲思路不讲操作" ✗"上一页讲过"

## 四、措辞平实,不抖机灵

- 禁俏皮口语收尾:✗"随你排" ✗"随手就算" ✗"看完不知道该干嘛"
- 禁判官句式:✗"……才叫 BI" ✗"……才算报表" → ✓平行陈述("图表是终点的,是制图;能继续追问的,是 BI")
- 禁指令式自检:✗"截完图问自己……" → ✓陈述原则("红色是信号,不是装饰")
- 结论可以有力,用词必须正;与 report-voice 的务实用词规范同源

## 五、例子密度(教学页的硬指标)

- **每条论点配具体例子或示例表**,海报式金句只能当收束,不能当正文
- 例子数字要**可验算、跨页可对账**(单价×数量=金额;同一演示世界观贯穿全片)
- 指标必须**真实存在**:不拿没人用的指标演道理(✗钢材水泥混算"吨均价" → ✓供应商准时交付率)
- 新概念**先类比后命名**:听众没听过的术语,先给生活类比再给名字(✓"交易表像银行流水,快照表像账户余额")

## 六、断言准确,范围讲清

- 不写过头断言:✗"BI 只吃一行一事实" → ✓"记录能算出结果,结果还原不回记录"
- 每条规则交代**适用范围**:✓"这三条说的是手工维护、要喂给 BI 的台账——给人看的汇报表不在此列"
- 区分"单一结论"与"分析主题":报表页围绕主题组织、可被追问,不是 PPT 式的一句死结论

## 自检清单(成稿后扫一遍)

- [ ] 有没有夸自己项目的统计数字?
- [ ] 有没有第二人称的"你做错了"句式、人际冲突描写?
- [ ] 有没有"接下来 / 回来收 / 上一页"类调度语?
- [ ] 有没有"才叫 / 随你 / 问自己"类用词?
- [ ] 每条论点旁边有例子或表格吗?数字能对账吗?
- [ ] 每条规则说清了管什么、不管什么吗?
