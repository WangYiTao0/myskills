# PPT 设计原则参考 (CRAP 四原则)

来源:《写给大家看的设计书》第 4 版,Robin Williams。
仅用于 PPT/演示文稿场景,与 `milwaukee-ppt` 配合使用。

---

## 核心思想

**优秀设计的秘密就是 4 个原则,知道名字就能说出问题。**

> "一旦能够说出什么东西的名字,就会很容易注意到它。" —— Robin Williams

看到一页 PPT 觉得不对劲,大多数人只能说"不好看",但说不出哪里不好。
本 skill 的目的:让你能**指着具体地方说出问题名字**。

### 与 milwaukee-ppt 的分工

| 范围            | 谁管                              |
| --------------- | --------------------------------- |
| Agenda 顺序     | milwaukee-ppt                     |
| PDCA 标签       | milwaukee-ppt                     |
| 标题三层结构    | milwaukee-ppt                     |
| 品牌红色用量    | milwaukee-ppt                     |
| **单页元素布局**| **本 skill**                      |
| **字体搭配**    | **本 skill**                      |
| **颜色组合判断**| **本 skill**                      |
| **视觉混乱诊断**| **本 skill**                      |

milwaukee-ppt 说"红色 ≤ 5%,只标关键数字 / AFTER / milestone";
本 skill 说"为什么红色锚点应该是对比而不是装饰"。

## 1. 亲密性 (Proximity) —— 相关的靠近,无关的分开

### 一句话

彼此相关的项**应当靠近,归在一组**。靠近 = 是一组;分开 = 不是一组。
眼睛看到靠近的东西自动判断为一个单元。

### 在 PPT 中怎么用

- 标题与它的副标题:贴近(行距 < 标题字号 1 倍)
- 章节标号与章节名:贴近,与正文拉开
- 图表与图表说明:贴近,与隔壁图表的说明拉开
- 数据 + 单位 + 注释:作为一组放在一起
- BEFORE / AFTER 对比:组内紧凑,组间拉开

### 要避免的问题

- 一页上有 5 个以上孤立元素(超过 3~5 个就该考虑分组)
- 元素之间留**等距空白**(等距 = 都是一组 = 都不是一组,等于没分)
- 不相关的东西因为"那里有空"就放在一起
- 标题离正文远,反而离上一节的图近 —— 视觉上归错组

### 自查

> 微微眯起眼睛,数页面上能看到几个"视觉块"?
> 超过 5 个就要想办法合并或分组。
> 每两个块之间的距离是否反映了内容的相关度?

### 典型修复

```
❌ Before: 标题   .   副标题   .   正文   .   图   .   图说
            (五个块均匀分布,看不出层级)

✅ After:  标题
           副标题      ←两行贴近=一组(标题组)

           正文段落... ←独立块

           图
           图说        ←两行贴近=一组(图组)
```

## 2. 对齐 (Alignment) —— 每个元素都要与另一个元素有视觉联系

### 一句话

**任何东西都不能随意安放**。每个元素必须能找到另一个元素与它对齐(共享一条看不见的线)。

### 在 PPT 中怎么用

- 一页内**优先用左对齐或右对齐**,不要默认居中
- 居中是新手最爱,因为"安全",但效果通常**乏味、业余**
- 同一页**不要混用多种对齐方式**(左对齐 + 居中 + 右对齐 = 业余感)
- 子标题、bullet、图说,要么都和正文左边对齐,要么都和右边对齐
- 数字列对齐:整数对小数点,不要左对齐数字

### 居中的两个例外

居中**不是绝对不能用**,但要有意为之:

1. **正式、庄重**的内容(封面、致谢页、引言)—— 居中合适
2. **明显居中**(放大、加空间、加强烈字体)—— 而不是 12pt Times 加两个回车的"默认居中"

> "如果想让文本居中,至少让它明显一点。" —— Robin Williams

### 要避免的问题

- 居中**当作默认**(初学者最大问题)
- 同页混用 2~3 种对齐方式
- 软边界(居中的元素彼此连接弱)代替硬边界(左/右对齐的硬线)
- 看似对齐但**差几像素** —— 对齐就要严格对齐,不要"差不多齐"

### 自查

> 在脑子里画几条垂直/水平线穿过页面,
> 每个元素都至少在一条线上吗?
> 这一页一共有几条对齐线?超过 3 条就太乱。

### 与 milwaukee-ppt 的连接

milwaukee-ppt 要求"标题三层结构"(主标题 + 副标题 + 正文 lead),
本 skill 补充:**三层必须左对齐成一条线**,不要主标题居中、副标题左对齐。

## 3. 重复 (Repetition) —— 视觉元素在整套作品中反复出现

### 一句话

设计中的某些视觉特征,要在整套作品中**反复出现**,从而把分散的页面拧成一个整体。

### 在 PPT 中怎么用

可以重复的元素:

- **颜色**(同一品牌红用在每一节的关键数字上)
- **字体组合**(标题用 A,正文用 B,贯穿整套)
- **图标风格**(全套统一线性或全套统一填充)
- **图表配色**(每个柱状图都用同一组色)
- **页眉/页脚装饰**(同一条细线、同一个位置)
- **章节起始页样式**(每个章节都一样的过渡页)

### 要避免的问题

- 重复**过度** —— 红色加在所有标题、所有数字、所有边框上 = 红色失去强调作用
- 重复**不足** —— 每一页都不一样,失去整体感
- 模板"自带"重复(页眉、页脚)以外,**没有自己设计的重复元素**

### 自查

> 把整套 PPT 缩成缩略图视图,
> 能不能一眼看出"这是同一套"?
> 有没有 2~3 个视觉特征贯穿始终?

### 与 milwaukee-ppt 的连接

milwaukee-ppt 已经给了一个强重复元素:**顶部红色 banner + 左上 logo**。
本 skill 提醒:再加 1~2 个自己设计的重复(比如所有页的关键数字都用同一种"大号粗体红色"格式),效果会更整体。

### 重要平衡:重复 vs 对比

重复让东西**连成一体**,对比让东西**分开层级**。两者**不冲突,共同作用**:
重复建立"一套"的感觉,对比在这一套内部建立强弱。

> "重复就像穿黑色晚礼服配红色高跟鞋 + 红色口红 + 一朵红色小花 —— 但不要再加红耳环、红手提包、红上衣。"

## 4. 对比 (Contrast) —— 要不同,就截然不同

### 一句话

**不要让两个元素"差不多"**。差不多 = 冲突;要么完全一样,要么截然不同。

### 在 PPT 中怎么用

对比可以来自:

- **大小**(标题 40pt vs 正文 14pt,比例 ≥ 3:1)
- **粗细**(细体 vs 粗体,不要 Regular vs Medium)
- **颜色**(深灰 vs 亮红,不要深灰 vs 中灰)
- **形状**(方块图标 vs 圆形头像)
- **方向**(横排标题 vs 竖排标号)
- **空间**(密集文字区 vs 大片留白)

### "字体对比 6 法"(书里专门一章)

如果两种字体放在一起不舒服,通常是因为它们**太像但又不一样**(冲突),不是真正的对比。
真正的字体对比靠这 6 种差异之一(或组合):

| 维度 | 例子 |
| ---- | ---- |
| 大小 | 24pt vs 72pt |
| 粗细 | Light vs Black |
| 结构 | 衬线体 vs 无衬线体(宋 vs 黑) |
| 形状 | 直体 vs 斜体,圆字 vs 方字 |
| 方向 | 横排 vs 竖排 |
| 颜色 | 深灰 vs 品牌红 |

**规则**:不要只对比一种,**结合 2~3 种**才有力。

### 要避免的问题(最重要)

- **犹豫不决** —— 想做对比但又怕"太跳",结果做出"略微不同" = 冲突,最糟
- 用粗线对比更粗的线、棕色对比黑色 —— **不够狠**
- 同页用 2 种相似的字体(都是黑体,但一个是雅黑一个是思源黑) —— 视觉冲突
- 在 BEFORE / AFTER 对比里给 BEFORE 也加红色 —— 红色锚点该专属 AFTER(milwaukee-ppt 也强调)

### 自查

> 这两个元素是想表达"层级不同"还是"内容并列"?
> 如果是层级不同,大小/粗细/颜色差异**够不够看出来**?
> 如果不够,不要小修小改,**加倍差异**。

### Robin 的硬规则

> "如果各项不完全一样,干脆让它们截然不同!"
> "Don't be a wimp." (不要畏畏缩缩。)

## 5. 字体使用 —— 协调 / 冲突 / 对比

### 三种关系

| 关系 | 定义 | 适用场景 |
| ---- | ---- | -------- |
| 协调 (Concord) | 同一种字体 / 同一字体家族 | 稳妥但可能乏味,适合正式场合 |
| 冲突 (Conflict) | 两种相似但不一样的字体 | **永远要避免** |
| 对比 (Contrast) | 两种截然不同的字体 | 最有力,但要 2~3 种维度差异 |

### PPT 中的字体建议

- **整套 PPT 用 2 种字体足够**(一种标题用,一种正文用)
- 标题字体和正文字体要在**结构维度**上不同(衬线 vs 无衬线最稳)
- 避免**两种无衬线体混用**(雅黑 + 思源黑 = 冲突)
- 同字体家族的不同字重可以一起用(Light + Bold 是对比,Regular + Medium 是冲突)
- 中文字体推荐组合:
  - 思源黑体(标题粗) + 思源宋体(正文) —— 黑+宋对比,稳
  - 思源黑体 Bold(标题) + 思源黑体 Light(正文) —— 同家族强对比
- **避免**:微软雅黑作所有用途 + 偶尔混进宋体作"装饰" —— 冲突

## 6. 颜色运用

颜色不是装饰,是**信息层级的工具**。Robin 的色轮哲学:**色轮是放心选色的工具,不是规则**。一旦掌握色轮上的几种关系,可以放心地用看似"怪异"的组合,因为它们在数学上就是和谐的。

### 6.1 色轮基础(12 色)

```
三原色 (Primary):     红、黄、蓝          —— 无法用其他颜色调出
三间色 (Secondary):   橙、绿、紫          —— 三原色两两等量混合
第三色 (Tertiary):    橙黄、橙红、紫红、    —— 间色与相邻原色再混合
                      紫蓝、蓝绿、黄绿
```

理解这 12 色的位置,后面的"颜色关系"就是从色轮上挑点的几何游戏。

### 6.2 五种颜色组合关系

| 关系 | 怎么选 | 视觉效果 | 适合场景 |
| ---- | ------ | -------- | -------- |
| **单色 (Monochromatic)** | 1 个色相 + 不同明度 | 最稳,绝不出错,可能乏味 | 公司汇报、严肃内容、单色品牌 |
| **类似色 (Analogous)** | 色轮相邻 2~3 色(如红+橙+黄) | 和谐,有暗示性(暖/冷季节感) | 季节性、情绪化内容 |
| **互补色 (Complementary)** | 色轮正对的两色(如红+绿,蓝+橙) | 对比最强,**一种主一种强调** | 数据图、AFTER 强调、CTA |
| **三色组 (Triadic)** | 色轮上等距三点(如红+黄+蓝) | 平衡有趣,但儿童化 | 教育、轻松场合;PPT 慎用 |
| **分裂互补 (Split-Complementary)** | 一色 + 其对面色的两侧色 | 比互补色更细腻 | 想要对比但不太刺眼 |

**关键洞见**:这 5 种关系是**安全网**。当你不知道选什么色,从色轮上挑一个关系,几乎不会出错。

### 6.3 暗色和亮色 —— 真正拓宽你的色板

到此为止只用了**纯色调** (Hue)。书里最有用的一点:**色调 + 黑 = 暗色 (shade);色调 + 白 = 亮色 (tint)**。

把 6.2 的 5 种关系**和暗色/亮色组合**:

```
纯红 + 纯绿        =  圣诞节,容易俗
深红 + 深绿        =  圣诞气息消失,变沉稳高级 ← 这是 PPT 想要的
浅红(粉) + 浅绿    =  春天感
深红 + 浅绿        =  对比加大,视觉层次更丰富
```

**Robin 的金句**:儿童产品最常用红黄蓝三原色,所以这种组合**幼稚到难以摆脱**;但只要换成它们的暗色和亮色,立刻变高级。

#### 在 PPT 中的实战意义

不要直接用 PowerPoint 默认色板里的**纯色调**,它们看起来要么刺眼要么幼稚。
取色板里同一色相的**深一档或浅一档**作为主色,瞬间专业感:

```
❌ #FF0000 纯红       —— 太刺眼,默认 PPT 红
✅ #DB011C Milwaukee Red  —— 加了一点黑,沉
✅ #B71C1C 深红           —— 暗色版,严肃场合
✅ #FFEBEE 浅粉           —— 亮色版,做强调框背景
```

### 6.4 色质 —— 颜色组合的隐形杀手

**色质 (Value)** = 颜色的明暗度。两种颜色色质太接近放在一起,会出现"视觉抖动",看起来模糊,文字读不清。

> "色质太接近,看上去会模糊不清。在复印机上印出来,文本就看不清。"

#### 自查方法(超简单)

把 PPT 转成**灰度**看一眼:

- 想强调的元素 vs 背景,色质差**够不够**?(灰度版能不能看出来?)
- 数据图表里多个数据系列,色质会不会都差不多变成一片灰?(数据图通常应该靠**色相差**而不是色质差,但**文字 vs 背景**则相反:必须靠色质差才能读清)

#### 在 PPT 中的实战意义

- **文字 vs 背景**:色质差必须够大(深灰文字 + 白底,或白字 + 深色底);**不要**用中灰文字配白底,或浅灰配中灰底
- **强调色 vs 主体色**:即使色相不同,如果色质相近(比如中度的红 + 中度的绿),强调效果会消失
- **数据图配色**:多个系列要色相不同 + 色质相近(看起来才像一组);但前景柱状图 vs 背景网格线,要色质拉开

### 6.5 暖色 vs 冷色 —— **永远不要均衡**

| 类别 | 包含 | 行为 |
| ---- | ---- | ---- |
| **暖色** | 红、橙、黄(以及偏红偏黄的灰、棕、米色) | **趋进**——立刻吸引眼球 |
| **冷色** | 蓝、绿、紫(以及偏蓝偏绿的灰) | **后退**——做背景 |

**Robin 硬规则**:暖色和冷色**绝不能均衡使用**。

- 想突出 = 用暖色,但**少量**(暖色一点点就够)
- 做背景或大面积 = 用冷色(可以大面积铺)
- 50/50 的暖冷色会让眼睛不知道看哪里,**整页失焦**

#### 在 PPT 中的实战意义

- **关键数字**用暖色(红/橙),**正文和图表**用冷灰
- **不要**把整页背景涂成暖色(整页红橙黄 = 不舒服 + 抢戏)
- **AFTER 用暖色,BEFORE 用冷色或灰** —— 这是 milwaukee-ppt 红色锚点规则的颜色学原理

### 6.6 PPT 配色决策流程

Robin 给了"如何取舍"的实用思路,搬到 PPT 场景:

1. **有品牌色吗?** → 用品牌色 + 它的暗色/亮色作色板(milwaukee-ppt 走这条)
2. **有照片或主视觉吗?** → 用滴管从照片里取色,选 1 个主色 + 1 个强调色
3. **有季节性或情绪倾向吗?** → 用类似色(夏=红黄、冬=蓝、秋=橙棕、春=绿)
4. **都没有,从零开始?** → 先做单色(灰阶 + 一个强调色),最稳

### 6.7 RGB vs CMYK —— PPT 一律用 RGB

| 模型 | 用于 | 原理 |
| ---- | ---- | ---- |
| **RGB** | 屏幕、PPT、Web、视频 | 光的混合,所有色相加 = 白 |
| **CMYK** | 印刷、画册、海报 | 油墨混合,所有色相加 = 黑 |

**PPT 几乎只在屏幕上看,一律 RGB**。

#### 但有两个 PPT 特有的颜色陷阱

1. **投影色偏**:投影仪通常会让**红色偏橙、蓝色偏紫、深色偏灰**;在自己屏幕上完美的配色,投出来可能完全变样。重要汇报前一定要**用目标投影仪过一遍**。
2. **环境光对比度**:亮室开会用 PPT,色质差必须比暗室更大;否则浅色看不见。设计时假设投影环境是"中等亮度",再加 20% 安全余量。

### 6.8 PPT 颜色实用规则(综合)

- **限制色相数量** —— 整套 PPT 用 **1~2 个色相**足够,靠明度(暗色/亮色)和灰阶做层级
- **深灰胜过纯黑** —— 正文用 `#1A1A1A` 比 `#000000` 柔和,长时间阅读不刺眼
- **强调色面积 ≤ 5%** —— milwaukee-ppt 规定的红色用量,普适规律
- **暖色前进、冷色后退、绝不均衡** —— 6.5 节核心
- **避免纯色调** —— PPT 默认色板里的纯红纯蓝直接换成它的暗色版
- **数据图配色**——同一组数据系列用**类似色**或**单色不同明度**,不要每根柱子一个色相(那是儿童产品配色)
- **可访问性兜底**——色盲友好:不要只靠红绿区分(用形状/标签辅助);文字与背景对比度 ≥ 4.5:1

### 6.9 与 milwaukee-ppt 的连接

milwaukee-ppt 规定的色板:

```
Milwaukee Red:    #DB011C   ← 这是"红色的暗色版"(加了黑),不是纯红 #FF0000
深灰文字:         #1A1A1A   ← 接近黑但有微暖
中灰副信息:       #6B6B6B   ← 拉开层级用
细分割线:         #E5E5E5   ← 极浅灰,几乎不抢戏
```

这本质上是**单色策略**(一个强调色 + 灰阶),按 6.2 表是"最稳、不出错"的方案。
本 skill 不增加颜色,只在判断时补两点:

1. 红色锚点的**位置**是否真的形成对比(色质 + 暖色趋进,只标关键数字 / AFTER / milestone)
2. 同页**红色面积**是否真的 ≤ 5%(milwaukee-ppt 规则的色彩学原理:暖色绝不均衡)

---

## 7. 综合诊断流程 —— 看到一页 PPT 觉得不对,按顺序问自己

1. **亲密性**:相关的东西靠近了吗?无关的东西分开了吗?数一下页面上能看到几个视觉块,超过 5 个就分组。
2. **对齐**:能画出几条对齐线?超过 3 种对齐方式(左/中/右混用)就乱。居中是不是默认而非选择?
3. **重复**:这页有没有承接整套的视觉特征(颜色/字体/装饰)?
4. **对比**:想表达层级的地方,差异够不够狠?是不是"差不多"而非"截然不同"?
5. **颜色**(转灰度自查):色质差够吗?暖冷是不是均衡了?强调色面积 ≤ 5% 吗?有没有用 PPT 默认的纯色调没改?

按这个顺序看一遍,大多数"看起来不对"的页都能定位到问题。

## 8. 与 milwaukee-ppt 的完整配合示例

假设要做一页 Milwaukee Tool 项目成果汇报页(第 06 节,PDCA = D/C):

milwaukee-ppt 给的约束:

- 顶部红 banner + 左上 logo
- 主标题 `项目行动与成果 / OUTCOMES`
- 副标题 action title,例如 `20+ 报表稳定运行,用户满意度 95%`
- PDCA 小图标 `D` 放在正文区右上角
- 红色 ≤ 5%,只标关键数字 / AFTER / milestone
- 禁止画饼词汇,所有动词已完成态

本 skill 在此基础上做单页视觉判断:

- **亲密性**:每个数字(20+ / 95% / 9 个月) + 它的标签 + 它的注释,作为一组,组间留大空白
- **对齐**:三组数据左对齐成一条线,标签贴在数字下方左对齐
- **重复**:三个数字用同一种"大号粗体红色"格式,贯穿整套 PPT
- **对比**:数字 60pt 红色,标签 14pt 深灰 —— 大小对比 ~4:1 + 颜色对比,够狠

如果三个数字大小相同 + 居中对齐 + 颜色都是黑色,即使内容符合 milwaukee-ppt 的所有规则,也会**看起来平淡**。本 skill 补的就是这一层。

## 10. 不在本 skill 范围内的内容

本 skill 是 PPT 视觉判断框架,不涉及:

- **内容结构 / Agenda 顺序** —— 看 milwaukee-ppt
- **PDCA 标注 / 标题三层结构** —— 看 milwaukee-ppt
- **品牌色规范** —— 看 milwaukee-ppt
- **PPT 文件生成 (.pptx 制作)** —— 看 pptx skill
- **网页/海报/印刷设计** —— 原则相通但执行不同,本 skill 仅覆盖 PPT
- **复杂图表设计 / 数据可视化** —— 本 skill 只讲基础原则,不涉及图表设计学
- **专业色彩管理 / ICC 色彩配置** —— 印刷工作流的色彩校准超出本 skill,PPT 用 RGB 即可
